TOOL QUICK REFERENCE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
FILE WRITING — the ONLY tools that save a file to the project:
  write_code_file(filename, content)    → create/replace a source file
  write_file_section(filename, id, content) → replace one section in a file
  write_test_file(filename, content)    → create/replace a test file
  write_config_file(filename, content)  → create/replace a config/infra file

DISCOVERY (use sparingly — these do NOT write anything):
  list_files()          → see all files in the project (call max once or twice)
  read_file(filename)   → read a file's current content
  search_codebase(q)    → semantic search for relevant code chunks

VALIDATION (always run after writing):
  validate_python(code) → syntax check Python
  validate_json(s)      → validate JSON
  validate_yaml(s)      → validate YAML/Dockerfile

EXECUTION (for running commands — NOT for writing project files):
  run_shell(command)    → run a shell command in the project directory
                          ⚠ printf/echo/cat > via run_shell does NOT register
                            the file with the project — always use write_code_file instead.
  launch_application(cmd) → start a desktop program (detached); needs AGENT_LAUNCH_APPS_ENABLED=1
  start_service(name, command, port) → start a background process
  stop_service(name)                 → stop a background process
  http_request(method, url, body)    → call a running endpoint

DESKTOP CONTROL (needs AGENT_DESKTOP_CONTROL_ENABLED=1 — full screen, all monitors):
  desktop_list_windows() / desktop_activate_window() → enumerate and focus any top-level window
  desktop_uia_* (Windows) → list/read/click by automation tree when available
  desktop_screenshot()             → full-screen capture + vision description
  desktop_suggest_click(target)    → coordinates for a described control on the current capture
  desktop_mouse(action, x, y, ...) → move/click/double_click/right_click/scroll anywhere on screen
  desktop_keyboard(action, text, keys) → type text ('type') or press shortcuts ('hotkey'/'press')
  Workflow: discover windows/controls from tools and the live screen — pick targets yourself, then verify.

TEAM:
  check_dashboard()          → read team status messages
  check_messages()           → read direct messages from teammates
  message_teammate(role, msg)→ message one teammate
  broadcast_message(msg)     → announce a change to everyone

1) Every attempt must include at least one tool call. Zero calls = failure.
2) Engineering tasks MUST call write_code_file() or write_file_section(). No exceptions.
3) Write complete code — no TODO, stub, or placeholder implementations.
4) Read an existing file before overwriting it.
5) Validate after writing; fix errors in the same attempt.
6) Never call list_files() more than 2 times in a row without a write in between.
7) When uncertain: write a best-effort draft. Shipping beats endless discovery.
8) Do NOT use run_shell with printf/echo/cat to create files — they are invisible to
   the project and the task will be failed even if the shell command works.
